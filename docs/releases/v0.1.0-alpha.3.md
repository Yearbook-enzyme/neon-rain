# Neon Rain v0.1.0-alpha.3

This release turns Neon Rain's experimental controls into a configurable visualizer experience with complete scenes, independent color palettes, remembered preferences, live configuration reload, and polished Linux installation tools.

## Highlights

- Six coordinated scenes: `classic-matrix`, `lucid-dream`, `cyber-tunnel`, `aurora-drift`, `ember-terminal`, and `redline`
- Independent theme and palette selection
- XDG configuration and remembered session state
- `F12` scene cycling and `F3` palette cycling
- `Home` live configuration reload
- `End` immediate session save
- `Insert` status-toast toggle
- On-screen status messages
- Improved helper discovery and Linux installer/uninstaller behavior
- Screenshot and desktop-video capture helper

## Run with Nix

```bash
nix run github:Yearbook-enzyme/neon-rain/v0.1.0-alpha.3
```

## Try a scene

```bash
nix run github:Yearbook-enzyme/neon-rain/v0.1.0-alpha.3 --   --scene lucid-dream   --no-media
```

List all scenes:

```bash
nix run github:Yearbook-enzyme/neon-rain/v0.1.0-alpha.3 -- --list-scenes
```

## Generic Linux archive

Download:

- `neon-rain-0.1.0-alpha.3-x86_64-linux.tar.gz`
- `neon-rain-0.1.0-alpha.3-x86_64-linux.tar.gz.sha256`

Then:

```bash
sha256sum -c neon-rain-0.1.0-alpha.3-x86_64-linux.tar.gz.sha256
tar -xzf neon-rain-0.1.0-alpha.3-x86_64-linux.tar.gz
cd neon-rain-0.1.0-alpha.3-x86_64-linux
./neon-rain --scene lucid-dream
```

## Configuration

Create the default editable configuration:

```bash
neon-rain --write-default-config
```

The example is scene-first. Optional theme, palette, FOV, flight, and cinematic overrides are commented so changing the scene line alone applies the full scene.

## Tested scope

- NixOS with KDE Plasma Wayland
- AMD Radeon RX 580 through Vulkan/RADV
- Native Ubuntu compilation and tests in GitHub Actions
- Generic Ubuntu 22.04-built `x86_64` Linux archive
- Empty temporary XDG configuration and state directories
- Interactive scenes, status toast, help overlay, and signal inspector

The generic archive still depends on host graphics drivers, desktop-session libraries, and Fontconfig. Broader real-machine Linux testing remains valuable.
